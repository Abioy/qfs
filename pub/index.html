Welcome to QFS!

Quantcast File System (QFS) is a high-performance, fault-tolerant, distributed file system developed to support MapReduce processing, or other applications reading and writing large files sequentially.

Obtaining QFS

Pre-built Binaries

Several binary distributions have been provided to make it easy to try QFS. Obtain the appropriate binary tarball from the downloads page and extract it as 

$ tar xvfz $TARBALL
to use the binaries.

Build Binaries From Source

It is also possible to obtain the QFS source (either via git clone or by downloading source tarball) and build the binaries. Build dependencies are:
cmake, make, git, g++, boost-devel, xfsprogs-devel, libuuid-devel, openssl-devel, jdk, java-devel, ant

The Developer Documentation has detailed information on build dependencies for various platorms. Once the dependencies are installed:

$ git clone https://github.com/quantcast/qfs.git
$ cd qfs
$ make

Using QFS

The QFS Wiki has a quick start guide to test-drive QFS on a single node.

For cluster-wide deployments (simple cluster or multi-rack cluster) and administration of QFS, navigate to Deployment Guide and Administrator's Guide in the QFS Wiki.

The Wiki also has details on performance benchmarks that compare QFS with other distributed file systems.
