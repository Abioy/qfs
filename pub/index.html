Welcome to QFS!

Quantcast File System (QFS) is a high-performance, fault-tolerant, distributed file system developed to support MapReduce processing, or other applications reading and writing large files sequentially.

Obtaining QFS

Pre-built Binaries

Several binary distributions have been provided to make it easy to try out QFS. Obtain the appropriate binary tarball from the downloads page and extract it as 

$ tar xvfz $TARBALL
to use the binaries.

Build Binaries From Source

One could also obtain the QFS source (either via git clone or downloading source tarball) and build the binaries. Build dependencies are:
cmake, make, git, g++, boost-devel, xfsprogs-devel, libuuid-devel, openssl-devel, jdk, java-devel, ant
The Developer Documentation has detailed information on build dependencies for various platorms. With the dependencies present, one could do

$ git clone https://github.com/quantcast/qfs.git
$ cd qfs
$ make

Using QFS

The QFS Wiki has a quick start guide for deploying QFS on a single-node.

For cluster-wide deployments (simple cluster or multi-rack cluster) and administration of QFS, please refer to the QFS Wiki links Deployment Guide and Administrator's Guide.

The Wiki also has details on the benchmarking work that we did comparing QFS with other distributed file systems.
